# SQL*Loader (sqlldr) examples

These files show how to populate the seed tables using SQL*Loader on a Linux host (e.g., your AWS EC2 instance).

## Prereqs

- Run SQL*Loader on a machine that has Oracle client tools installed (`sqlldr`). On many Oracle DB servers, it’s already available.
- The CSV files must be on the same machine where you run `sqlldr`.
- Use the **schema owner** account (the user that owns the tables created by `@scripts/install.sql`).

## Run

From this folder:

```bash
cd scripts/06_data/sqlldr

# CUSTOMER
sqlldr userid=<SCHEMA_USER>/<PASSWORD> control=LoadCUSTOMER.ctl log=LoadCUSTOMER.log bad=LoadCUSTOMER.bad discard=LoadCUSTOMER.dsc

# RESTAURANT
sqlldr userid=<SCHEMA_USER>/<PASSWORD> control=LoadRESTAURANT.ctl log=LoadRESTAURANT.log bad=LoadRESTAURANT.bad discard=LoadRESTAURANT.dsc

# MENU_ITEM
sqlldr userid=<SCHEMA_USER>/<PASSWORD> control=LoadMENU_ITEM.ctl log=LoadMENU_ITEM.log bad=LoadMENU_ITEM.bad discard=LoadMENU_ITEM.dsc

# DRIVER
sqlldr userid=<SCHEMA_USER>/<PASSWORD> control=LoadDRIVER.ctl log=LoadDRIVER.log bad=LoadDRIVER.bad discard=LoadDRIVER.dsc
```

## Notes

- These control files load explicit IDs into `..._id` identity columns. This works because your schema uses `GENERATED BY DEFAULT ON NULL AS IDENTITY`.
- `created_at` is omitted for `CUSTOMER` so Oracle uses the default `SYSDATE`.
- If you re-run loads without clearing tables, you may hit PK/unique constraint errors.
